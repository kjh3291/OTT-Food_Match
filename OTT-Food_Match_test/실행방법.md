# OTT-Food_Match 단위 테스트 실행 방법

메인 코드(`OTT-Food_Match/`)는 전혀 수정하지 않고, 이 `OTT-Food_Match_test/` 폴더 안에서만
순수 함수를 단위 테스트하고 커버리지를 측정합니다.

## 0. 사전 준비 (공통)

- Node.js LTS 설치: https://nodejs.org
- 설치 확인:
  ```
  node -v
  npm -v
  ```

## 1. macOS (터미널)

```bash
# 1) 이 폴더로 이동 (경로는 실제 위치에 맞게)
cd ~/Downloads/OTT-Food_Match_test

# 2) 의존성(Jest) 설치 — 최초 1회
npm install

# 3) 테스트 실행
npm test

# 4) 커버리지 측정
npm run coverage
```

## 2. Windows (PowerShell 또는 명령 프롬프트)

```powershell
:: 1) 이 폴더로 이동 (경로는 실제 위치에 맞게)
cd C:\Users\사용자명\Downloads\OTT-Food_Match_test

:: 2) 의존성(Jest) 설치 — 최초 1회
npm install

:: 3) 테스트 실행
npm test

:: 4) 커버리지 측정
npm run coverage
```

팁: 폴더에서 `Shift + 마우스 우클릭 → "여기에 터미널 열기 / PowerShell 창 열기"`를
쓰면 `cd` 없이 바로 명령을 입력할 수 있습니다.

## 3. 결과 확인

- `npm test` : 모든 테스트의 통과/실패가 터미널에 출력됩니다.
- `npm run coverage` : 아래와 같은 커버리지 표가 출력되고,
  ```
  ----------------------|---------|----------|---------|---------|
  File                  | % Stmts | % Branch | % Funcs | % Lines |
  ----------------------|---------|----------|---------|---------|
  ```
  추가로 `coverage/lcov-report/index.html` 파일이 생성됩니다.
  이 HTML을 브라우저로 열면 줄 단위로 어떤 코드가 테스트됐는지 시각적으로 볼 수 있습니다.

## 4. 특정 파일만 테스트하기 (선택)

```bash
npx jest movieUtils.test.js
npx jest --coverage movie-utils.js
```

## 5. 폴더 구성

- 원본에서 "순수 로직만" 복사해 CommonJS(module.exports)로 만든 모듈 + 그 테스트 파일로 구성됩니다.
- 네트워크(Firebase/TMDB), DOM 렌더링, 카카오맵 SDK, 브라우저 위치 등은
  단위 테스트 대상에서 제외했습니다 (자세한 이유는 README.md 참고).
